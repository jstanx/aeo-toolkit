# Answer-Block Checklist

Run each key section through this before publishing. Aim for all boxes checked.

## Per section
- [ ] The heading is the natural-language **question** a person would ask an assistant.
- [ ] A **direct answer** appears immediately after the heading (no warm-up).
- [ ] The answer block is **40–60 words** and states the answer in the first sentence.
- [ ] The answer is **self-contained** — it makes sense lifted out of the page.
- [ ] No back-references ("as above", "as mentioned", undefined "this/that/it").
- [ ] The answer is **specific** (names, numbers, conditions) and **accurate**.
- [ ] The section covers **one** question only.
- [ ] Supporting detail follows the answer; the section ends cleanly.

## Per page
- [ ] The single most important answer (plus a short summary) is **near the top**.
- [ ] No critical, citable fact is stranded in the **deep middle** of the page.
- [ ] Sections are ordered by a natural flow of the reader's questions.
- [ ] Paragraphs are short (2–4 sentences); lists/tables used for steps and comparisons.
- [ ] Headings are descriptive and do not skip levels.
- [ ] Questions/answers map back to the foundation **prompt set**.

## Hand-off
- [ ] Evidence added (statistics, quotes, sources) via **evidence-and-citations**.
- [ ] Markup added (Article/Person/FAQPage as relevant) via **structured-data**.
- [ ] Crawlability confirmed (content readable without JS) via **ai-retrievability**.
